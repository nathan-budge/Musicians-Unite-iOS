MeterWorx PitchDetector SDK

Overview

The MeterWorx™ PitchDetector software development kit enables simple, easy and precise pitch detection from a variety of audio source material.

For a quick usage example see Tutorial page.

Requirements

  • tools and environment:
    • Android:
      • API level 10
      • NDK r10d or later
      • GCC 4.7 or later or Clang 3 or later
      • x86 (SSSE3 or later) or ARM (v6 or later) CPU
      • does not itself link with any STL
    • iOS:
      • iOS 5.1.1 or later
      • Clang 3 or later
      • ARMv7 or later CPU
      • links with libc++
      • requires linking with the Accelerate framework (the development build additionally requires the Foundation framework)
    • OS X:
      • OS X 10.6 or later
      • Clang 3 or later
      • x86 SSE3 capable CPU
      • links with libstdc++
      • requires linking with the Accelerate framework (the development build additionally requires the CoreServices framework)
    • Windows:
      • Windows XP SP3 or later
      • Visual Studio: 2010 SP1 to 2013 (single library supports linking with all the supported versions, express editions are fine)
      • x86 SSE2 capable CPU
      • links with the static MSVC runtime
  • 3rd party libraries: none.

Build configuration

  • The include directory has to be added to the compiler's include path.
  • The libs directory has to be added to the linker's search path.
  • Two SDK builds are provided: the "release" build intended for public release/retail/RTM builds, and the "development" build that has basic sanity checks compiled in that will show assertion failures in cases of incorrect usage (depending on the target OS, assertion failures will be reported with one or more of the following: message boxes, stderr messages, system log entries and aborting or breaking into the debugger)
  • Code specifically optimised for a given architecture/CPU is provided in the form of separate libraries, no built in runtime detection and dispatching is provided. The following builds are provided:
    • Android:
      • ARMv5TE_soft-float (default/minimum guaranteed armeabi architecture) - NOT SUPPORTED!
      • ARMv6_VFP2 (only usable for offline processing)
      • ARMv7a_VFP3-D16 (default/minimum guaranteed armeabi-v7a architecture)
      • ARMv7a_NEON
      • ARMv8a_64 (default/minimum guaranteed arm64-v8a architecture)
      • x86-32_SSSE3 (default/minimum guaranteed x86 architecture)
      • x86-64_SSE4.2 (default/minimum guaranteed x86-64 architecture)
      • mips - NOT SUPPORTED!
    • iOS: single universal fat static library with armv7, armv7s, arm64, i386 (simulator) and x86_64 (simulator) slices
    • OSX:
      • x86-32_SSE3 + x86-64_SSSE3 universal static library
      • x86-32_SSE4.1 + x86-64_SSE4.1 universal static library
    • Windows:
      • x86-32_SSE2
      • x86-32_SSE4.1
      • x86-64_SSE3
      • x86-64_SSE4.1
    • for additional support and/or optimisations for specific architectures please contact us.

General usage and design notes

The LE MeterWorx™ SDK:

  • Uses two-phase initialisation. Public, no-fail constructors and destructors allow creation of stack-based objects and setup() member functions allow for dynamic (re)initialisation of created objects based on desired parameters. After an object is created, its setup() member function must be called at least once before it can be used.
  • Is designed to group and isolate operations that can potentially fail and to offer the no-fail guarantee wherever possible thereby simplifying error reporting. It is completely exception free and does not even use special error codes: in the few required places errors are reported through simple boolean values (errors always indicate memory allocation failures).
  • Does not handle incorrect usage. Incorrect usage or failure to comply with documented preconditions for a given function results in undefined behaviour or assertion failures in release or development builds, respectively.

Demo limitation

  • Demo builds are limited to approximately 15 seconds of valid measurements, after that zero is returned for each call to PitchDetector::pitch().
